# 安全策略

## 报告安全漏洞

如果您发现本项目存在安全漏洞，请通过以下方式报告：

### 📧 联系方式

- **电子邮件**: your-email@example.com
- **GitHub Security Advisories**: [创建安全公告](https://github.com/YOUR_USERNAME/study2026/security/advisories)

### 📋 报告内容

请提供以下信息：

1. **漏洞描述** - 详细说明漏洞的性质
2. **影响版本** - 受影响的版本号
3. **重现步骤** - 如何重现该漏洞
4. **潜在影响** - 漏洞可能造成的影响
5. **修复建议** - 如有，提供修复建议

### ⏰ 响应时间

- **初步响应**: 48 小时内
- **修复计划**: 7 个工作日内
- **公开披露**: 修复后 30 天

|------|----------|

| Previous | ✅ 支持 |
| Older | ❌ 不再支持 |

## 最佳实践

### 开发者

- 定期更新依赖
- 使用依赖检查工具（如 `npm audit`, `safety`）
- 遵循安全编码规范
- 实施代码审查

### 用户

- 使用最新稳定版本
- 不要在公开场合分享 API 密钥
- 定期更改密码
- 启用双因素认证（如适用）
