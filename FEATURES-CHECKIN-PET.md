# 🎉 用户留存功能 - 每日签到 + 学习伴侣

## 功能概述

为了留住访问用户，项目新增了两个核心留存功能：

### 1. 📅 每日签到系统

**核心机制：**
- 用户每日签到获得连续天数（streak）
- 连续签到获得徽章奖励
- 签到日历可视化展示
- 损失厌恶心理：中断会失去连续记录

**奖励机制：**
| 连续天数 | 奖励 |
|---------|------|
| 3 天 | 初露锋芒 |
| 7 天 | 周勤学者徽章 |
| 14 天 | 半月达人徽章 |
| 30 天 | 月勤学者徽章 |
| 100 天 | 百日传奇徽章 |

**访问页面：** `/checkin`

---

### 2. 🐾 AI 学习伴侣养成

**核心机制：**
- 每个用户拥有一只虚拟学习宠物
- 宠物随学习时间升级进化
- 宠物会主动发送消息提醒学习
- 情感连接：宠物需要喂食、玩耍、休息

**宠物等级：**
| 等级 | 名称 | 图标 | 所需经验 |
|-----|------|------|---------|
| 1 | 蛋蛋 | 🥚 | 0 |
| 2 | 小雏 | 🐣 | 100 |
| 3 | 学童 | 🐤 | 300 |
| 4 | 学子 | 🐥 | 600 |
| 5 | 学师 | 🦅 | 1000 |
| 6 | 学尊 | 🦉 | 1600 |
| 7 | 学圣 | 🐉 | 2500 |
| 8 | 学神 | ✨ | 4000 |

**互动方式：**
- 🍗 喂食：+15 心情，+5 经验
- 🎮 玩耍：+20 心情，+10 经验
- ☕ 休息：+30 精力，+5 经验
- 📚 学习：+20 经验

**访问页面：** `/pet`

---

## 技术实现

### 后端 API

#### 签到 API
```
POST   /api/v1/checkin              # 用户签到
GET    /api/v1/checkin/info         # 获取签到信息
GET    /api/v1/checkin/stats        # 全局统计
```

#### 宠物 API
```
GET    /api/v1/pet                  # 获取宠物信息
POST   /api/v1/pet/create           # 创建宠物
POST   /api/v1/pet/exp              # 增加经验
POST   /api/v1/pet/interact         # 互动
POST   /api/v1/pet/checkin-bonus    # 签到奖励
GET    /api/v1/pet/types            # 宠物类型列表
```

### 前端页面
- `apps/web/src/app/checkin/page.tsx` - 签到页面
- `apps/web/src/app/pet/page.tsx` - 宠物页面
- `apps/web/src/components/Sidebar.tsx` - 侧边栏导航（新增入口）

### 后端服务
- `apps/api/app/services/checkin_service.py` - 签到服务
- `apps/api/app/services/pet_service.py` - 宠物服务

## 心理机制

### 1. 习惯养成
- 每日签到形成习惯
- 连续记录不愿中断

### 2. 情感连接
- 宠物陪伴增加情感投入
- 宠物升级带来成就感

### 3. 即时反馈
- 每次互动都有视觉反馈
- 升级动画带来惊喜感

### 4. 损失厌恶
- 中断签到会失去连续记录
- 宠物心情低会"不开心"

## 使用方式

### 1. 访问签到页面
从侧边栏点击「每日签到」或访问 `/checkin`

### 2. 访问宠物页面
从侧边栏点击「学习伴侣」或访问 `/pet`

### 3. 与宠物互动
- 点击互动按钮进行喂食、玩耍、休息、学习
- 宠物会根据状态发送不同消息
- 学习时会获得经验值并可能升级
