# 🔄 工作流系统完整文档

## 📋 功能概览

Study2026 工作流系统是一个可视化的自动化流程编排工具，支持：

- ✅ 可视化节点编辑
- ✅ 4 种节点类型（触发器、动作、条件、API）
- ✅ 4 个预置模板
- ✅ 执行历史和监控
- ✅ 节点配置面板
- ✅ 拖拽式操作

---

## 🎨 节点类型

### 1. 触发器节点 (Trigger)
绿色节点，用于启动工作流

| 触发器 | 说明 | 配置参数 |
|--------|------|----------|
| 定时触发 | 按指定时间触发 | `time`: "09:00" |
| Webhook | HTTP 请求触发 | `path`: "/api/data" |
| 进度更新 | 学习进度变化触发 | `threshold`: 50 |

### 2. 动作节点 (Action)
蓝色节点，执行具体操作

| 动作 | 说明 | 配置参数 |
|------|------|----------|
| 发送邮件 | 发送电子邮件 | `template`: "daily_reminder" |
| 发送通知 | 推送通知 | `channel`: "email" |
| 生成报告 | 生成学习报告 | `format`: "html" |

### 3. 条件节点 (Condition)
橙色节点，条件分支判断

| 配置 | 说明 |
|------|------|
| `operator`: ">=" | 比较运算符 |
| `value`: 50 | 比较值 |

### 4. API 节点 (API)
紫色节点，调用外部 API

| 配置 | 说明 |
|------|------|
| `method`: "GET" | HTTP 方法 |
| `endpoint`: "/api/progress" | API 端点 |

## 📦 预置模板

### 1. 每日学习提醒
- **用途**: 每天早上 9 点发送学习提醒
- **节点**: 定时触发 → 发送邮件
- **执行频率**: 每天一次

### 2. 里程碑通知
- **用途**: 学习进度达到里程碑时发送通知
- **节点**: 进度更新 → 判断条件 → 发送通知
- **触发条件**: 进度 >= 50%

### 3. 周报生成
- **用途**: 每周日生成学习周报并发送
- **节点**: 每周触发 → 获取进度 → 生成报告 → 发送邮件
- **执行频率**: 每周一次

### 4. API 集成工作流
- **用途**: 连接第三方 API 获取数据
- **节点**: Webhook → 调用 API → 处理数据
- **适用场景**: 数据同步、自动化集成

## 🔧 使用指南

### 创建工作流

1. **选择模板或从头开始**
   - 点击"模板库"标签
   - 选择适合的模板点击"使用此模板"
   - 或直接在编辑器中添加节点

2. **添加节点**
   - 从左侧节点库拖拽节点到画布
   - 或点击节点按钮自动添加

3. **连接节点**
   - 鼠标悬停在节点右侧的连接点
   - 拖拽到目标节点的左侧连接点

4. **配置节点**
   - 点击节点打开配置面板
   - 填写参数
   - 点击保存

5. **保存和运行**
   - 点击右上角"保存"按钮
   - 点击"运行"测试工作流

### 执行历史

点击"执行历史"标签查看：
- 执行状态（成功/失败/运行中）
- 执行时间
- 执行耗时
- 点击"查看详情"查看执行日志

## 📊 API 端点

### 工作流管理

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/v1/workflows` | 获取工作流列表 |
| POST | `/api/v1/workflows` | 创建工作流 |
| GET | `/api/v1/workflows/{id}` | 获取工作流详情 |
| POST | `/api/v1/workflows/{id}/execute` | 执行工作流 |
| DELETE | `/api/v1/workflows/{id}` | 删除工作流 |
| GET | `/api/v1/workflows/{id}/executions` | 获取执行历史 |

### 请求示例

#### 创建工作流
```bash
curl -X POST http://localhost:8001/api/v1/workflows \
  -H "Content-Type: application/json" \
  -d '{
    "name": "我的工作流",
    "nodes": [...],
    "edges": [...]
  }'
```

#### 执行工作流
```bash
curl -X POST http://localhost:8001/api/v1/workflows/1/execute \
  -H "Content-Type: application/json" \
  -d '{"data": "test"}'
```

## 🎯 最佳实践

### 1. 命名规范
- 工作流名称：清晰描述用途
- 节点名称：体现功能
- 例如："每日学习提醒" 比 "工作流 1" 更好

### 2. 错误处理
- 在关键节点后添加条件判断
- 配置失败通知
- 定期检查执行历史

### 3. 性能优化
- 避免过长的执行链
- 使用异步动作
- 合理设置触发频率

### 4. 测试策略
- 先测试单个节点
- 再测试完整流程
- 监控生产环境执行

## 🛠️ 故障排除

### 问题 1: 节点无法连接
**解决**: 确保从输出点（右侧）拖到输入点（左侧）

### 问题 2: 工作流无法执行
**解决**: 
1. 检查是否有触发器节点
2. 检查节点配置是否完整
3. 查看执行历史错误信息

### 问题 3: 配置不保存
**解决**: 确保点击了"保存"按钮再离开页面
