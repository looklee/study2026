# 🔄 可视化工作流编辑器使用指南

支持节点连线、AND/OR 逻辑的工作流编排工具。

---

## 🎨 功能特点

### 1. 可视化连线
- ✅ 节点之间用线连接
- ✅ 带箭头的连接线
- ✅ 平滑的曲线连接（smoothstep）
- ✅ 拖拽式连接

### 2. 逻辑关系
- ✅ **AND (与)** - 默认模式，所有节点按顺序执行
- ✅ **OR (或)** - 任一节点触发即可执行

### 3. 节点类型（6 种）
| 类型 | 颜色 | 说明 |
|------|------|------|
| 🟢 触发器 | 绿色 | 启动工作流 |
| 🔵 动作 | 蓝色 | 执行操作 |
| 🟠 条件 | 橙色 | 条件分支判断 |
| 🟣 API | 紫色 | 调用外部 API |
| 🔵 LLM | 靛蓝色 | 调用大模型 |
| 🟢 数据 | 青色 | 数据操作 |

---

## 🎯 使用教程

### 1. 添加节点
1. 从左侧节点库选择节点类型
2. 点击节点按钮添加到画布
3. 拖拽节点调整位置

### 2. 连接节点
1. 鼠标悬停在节点的连接点（右侧输出或左侧输入）
2. 拖拽到目标节点的连接点
3. 自动创建带箭头的连接线

### 3. 配置逻辑
1. 顶部选择 **AND** 或 **OR** 模式
2. AND：所有节点按顺序执行
3. OR：任一节点触发即可

### 4. 配置节点
1. 点击节点选中
2. 右侧面板显示配置
3. 修改节点名称和参数
4. 点击保存

## 📊 节点说明

### 触发器（绿色）
- **定时触发** - 按指定时间触发
- **Webhook** - HTTP 请求触发
- **进度更新** - 学习进度变化触发

### 动作（蓝色）
- **发送邮件** - 发送电子邮件
- **发送通知** - 推送通知
- **生成报告** - 生成学习报告

### 条件（橙色）
- **判断条件** - 条件分支判断
- 有两个输出：是（绿色）/ 否（红色）

### API（紫色）
- **HTTP 请求** - 调用外部 API

### LLM（靛蓝色）
- **调用 LLM** - 调用大语言模型

### 数据（青色）
- **保存数据** - 保存到数据库

---

## 🔗 连接规则

### 输入/输出点
- **左侧** - 输入点（白色圆点）
- **右侧** - 输出点（白色圆点）
- **条件节点** - 右侧有两个输出点（是/否）

### 连接方向
- 从左到右连接
- 触发器 → 动作/LLM/数据
- 条件 → 动作（根据结果）

```
[定时触发 09:00] → [发送邮件]
```
- 逻辑：AND

- 节点：4
- 连接：3

## ⚙️ 配置说明

### 逻辑模式

**AND (与)**
- 所有节点按顺序执行
- 适合线性流程
- 默认模式

**OR (或)**
- 任一节点触发即可
- 适合并行触发
- 灵活模式

### 节点配置

每个节点可以配置：
- 节点名称
- 触发条件
- 执行参数
- 输出格式

## 🎯 最佳实践

### 1. 命名规范
- 节点名称清晰描述功能
- 例如："发送学习提醒" 比 "动作 1" 更好

### 2. 布局建议
- 从左到右排列
- 保持适当间距
- 避免连线交叉

### 3. 错误处理
- 在关键节点后添加条件判断
- 配置失败通知
- 定期检查执行历史

## 🔧 故障排除

### 问题 1: 无法连接节点
**解决**: 确保从输出点（右侧）拖到输入点（左侧）

### 问题 2: 工作流无法执行
**解决**:
1. 检查是否有触发器节点
2. 检查节点配置是否完整
3. 查看执行历史错误信息

### 问题 3: 配置不保存
**解决**: 确保点击了"保存"按钮再离开页面
